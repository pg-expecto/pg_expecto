# Отчёты и мониторинг цепи Маркова

## Содержание

1. **[mchain_reliability_report](#1-mchain_reliability_report)** – формирует расширенный отчёт о достоверности прогнозов с рейтингом 0–5 на основе объёма данных, стабильности и покрытия состояний.
2. **[mchain_incident_transitions_report](#2-mchain_incident_transitions_report)** – анализирует переходы в аварийные состояния (`correlation<0, os_trend=-1`) за указанный период, сравнивая сырую историю и текущую модель с забыванием.
3. **[mchain_summary_report](#3-mchain_summary_report)** – сводный отчёт, объединяющий конфигурацию, текущее состояние, достоверность прогнозов и анализ аварийных переходов.
4. **[mchain_incident_state_detail_report](#4-mchain_incident_state_detail_report)** – детальный отчёт по каждому аварийному состоянию (`correlation<0, os_trend=-1, wait_trend=1`) с топ-3 предшественников и статистикой инцидентов производительности.
5. **[mchain_health_check](#5-mchain_health_check)** – быстрая диагностика для систем мониторинга: возвращает OK/WARNING/CRITICAL с сообщением.
6. **[mchain_state_transition_matrix_report](#6-mchain_state_transition_matrix_report)** – текстовая матрица переходов между укрупнёнными группами состояний (9 или 27 групп) с возможностью взвешенного усреднения.

---

## 1. mchain_reliability_report

**Назначение**  
Формирует расширенный текстовый отчёт о достоверности прогнозов модели на основе трёх ключевых метрик: объём накопленных данных, стабильность вероятностей переходов и покрытие частых состояний. Итоговый рейтинг от 0 (полностью недостоверен) до 5 (максимально достоверен).

**Сигнатура**  
```sql
mchain_reliability_report()
RETURNS TEXT
```

**Входные параметры**  
Нет.

**Возвращаемое значение**  
Многострочный текст (TEXT) с отчётом, содержащим:

- Общий рейтинг достоверности (0–5) и его интерпретацию.
- Общее число переходов (`total_transitions`) и порог `min_transitions_for_forgetting` из конфигурации.
- Максимальное изменение вероятностей за последние две недели (`max_prob_change`) – показатель стабильности.
- Покрытие частых состояний (`coverage_pct`) – доля состояний с частотой >1%, для которых накоплено не менее 50 переходов.
- Рекомендации по улучшению модели или её использованию.

**Логика работы** (соответствует коду)

1. **Объём данных** – оценивает `total_transitions`:
   - `< 100` → рейтинг 0
   - `100–499` → рейтинг 1
   - `500–4999` → рейтинг 2
   - `≥ 5000` → базовая часть рейтинга = 3 (далее добавляются бонусы)

2. **Стабильность** – сравнивает вероятности переходов за две последовательные недели (`now()-14d … now()-7d` и `now()-7d … now()`):
   - Изменение < 2% → бонус +2
   - Изменение < 5% → бонус +1
   - Иначе → бонус 0

3. **Покрытие частых состояний** – для состояний, встречающихся в более чем 1% всех переходов, проверяет, имеют ли они минимум 50 наблюдений:
   - Покрытие ≥ 90% → бонус +1
   - Иначе → бонус 0

4. Итоговый рейтинг = базовая оценка + бонусы (максимум 5).

**Пример вызова**
```sql
SELECT mchain_reliability_report();
```

**Пример вывода (фрагмент)**
```
ОТЧЁТ О ДОСТОВЕРНОСТИ ПРОГНОЗОВ ЦЕПИ МАРКОВА
--------------------------------------------------------------------

1. ОБЩИЙ РЕЙТИНГ ДОСТОВЕРНОСТИ (0-5): 4
   Интерпретация: хорошая достоверность. Прогнозам можно доверять в большинстве ситуаций.

--------------------------------------------------------------------

2. ДЕТАЛИЗАЦИЯ ПО МЕТРИКАМ

   Общее число переходов (total_transitions): 12500
   ...
```

## 2. mchain_incident_transitions_report

**Назначение**  
Анализирует переходы в **аварийные состояния** за заданный временной интервал.  
**Определение аварийного состояния в этой функции:** `correlation < 0 AND os_trend = -1` (тренд ожиданий не учитывается).  
Отчёт объединяет:

- Сырую статистику из журнала `transition_log` (фактическая частота аварий).
- Взвешенные частоты из таблицы `markov_frequencies` (текущая модель, учитывающая забывание).

Позволяет сравнить, насколько модель с забыванием соответствует реальной истории.

**Сигнатура**  
```sql
mchain_incident_transitions_report(
    p_start TIMESTAMPTZ DEFAULT NULL,
    p_end   TIMESTAMPTZ DEFAULT NULL
)
RETURNS TEXT
```

**Входные параметры**  
| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|-----------|
| `p_start` | TIMESTAMPTZ | `now() - interval '7 days'` | Начало анализируемого периода |
| `p_end`   | TIMESTAMPTZ | `now()` | Конец периода |

**Возвращаемое значение**  
Текстовый отчёт, содержащий:

- Период анализа.
- Общее количество переходов и число переходов в аварию (с процентной долей).
- Количество различных аварийных состояний, в которые были переходы.
- Суммарный вес переходов в модели `markov_frequencies` и долю аварийного веса.
- Сравнение доли аварий в сырых данных и в модели (разница, интерпретация).
- Динамику по часам (если период ≤ 2 дней) или по дням (если период длиннее).
- Рекомендации в зависимости от доли аварий и расхождения с моделью.

**Логика работы**  
- Определяет список аварийных состояний через `state_descriptions` (`correlation < 0 AND os_trend = -1`).
- Агрегирует `transition_log` за указанный период.
- Вычисляет суммарный вес из `markov_frequencies` и вес, приходящийся на аварийные переходы.
- Формирует динамические срезы (почасовые или подневные) в зависимости от длины периода.
- Сравнивает проценты и выдаёт диагностику.

**Пример вызова**
```sql
SELECT mchain_incident_transitions_report();
SELECT mchain_incident_transitions_report('2025-05-01', '2025-06-01');
```

## 3. mchain_summary_report

**Назначение**  
Сводный отчёт, объединяющий информацию о достоверности прогнозов, анализе аварийных переходов, текущей конфигурации модели и состоянии системы. Быстро оценивает «здоровье» всей цепи Маркова.

**Сигнатура**  
```sql
mchain_summary_report(
    p_start TIMESTAMPTZ DEFAULT NULL,
    p_end   TIMESTAMPTZ DEFAULT NULL
)
RETURNS TEXT
```

**Входные параметры**  
Аналогичны `mchain_incident_transitions_report`.  
**Примечание:** отчёт использует определение аварийных состояний из вызываемой `mchain_incident_transitions_report` (т.е. `correlation < 0 AND os_trend = -1`).

**Возвращаемое значение**  
Текст, включающий:

1. **Конфигурацию модели** (флаги, alpha, half_life, интервалы, пороги).
2. **Текущее состояние системы** (`state_id`, параметры, прогноз риска на 1 минуту).
3. **Отчёт о достоверности прогнозов** (полный вывод `mchain_reliability_report`).
4. **Анализ переходов в аварию за период** (полный вывод `mchain_incident_transitions_report`).
5. **Общие рекомендации** на основе рейтинга достоверности и настроек забывания.

**Пример вызова**
```sql
SELECT mchain_summary_report(now() - interval '30 days', now());
SELECT mchain_summary_report();  -- период аварий = 7 дней
```

## 4. mchain_incident_state_detail_report

**Назначение**  
Детальная информация **по каждому аварийному состоянию**, в которое были переходы за указанный период.  

**Определение аварийного состояния в этой функции:**  
`correlation < 0 AND os_trend = -1 AND wait_trend = 1`.  

Для каждого такого состояния показывает:

- Сырую частоту переходов (из `transition_log`).
- Текущий вес в модели (из `markov_frequencies` с учётом забывания, **но берётся произвольная строка** – см. ограничение).
- Топ-3 предшествующих состояния (из каких состояний чаще всего приходили) с количеством переходов и процентной долей.

**Дополнительно:** если существует таблица `performance_incident`, в отчёт включается статистика по инцидентам производительности за тот же период (всего инцидентов, незавершённые, распределение по приоритетам, длительность, задержка после аварийных переходов).

**Сигнатура**  
```sql
mchain_incident_state_detail_report(
    p_start TIMESTAMPTZ DEFAULT NULL,
    p_end   TIMESTAMPTZ DEFAULT NULL
)
RETURNS TEXT
```

**Входные параметры**  
Аналогичны предыдущим отчётам.

**Возвращаемое значение**  
Текстовый отчёт, содержащий:

- Статистику по инцидентам производительности (если таблица существует).
- Для каждого аварийного состояния, имевшего переходы за период:
  - Идентификатор (`state_id`), корреляцию, тренд ожиданий (`wait_trend`).
  - Сырое количество переходов.
  - Вес в модели (произвольная строка из `markov_frequencies`).
  - Список до 3 предшествующих состояний с их параметрами, количеством переходов и долей.

Если переходов в аварийные состояния не было – выводится соответствующее сообщение.

**Ограничения**  
- Поле «Вес в модели» получается через `SELECT frequency FROM markov_frequencies WHERE to_state = ... LIMIT 1`, что **не отражает суммарный вес** всех входящих переходов в данное состояние. Для корректного анализа следует использовать `SUM(frequency)` по всем `from_state`.

**Пример вызова**
```sql
SELECT mchain_incident_state_detail_report(now() - interval '30 days', now());
```

**Пример вывода (фрагмент)**
```
ДЕТАЛЬНЫЙ ОТЧЁТ ПО АВАРИЙНЫМ СОСТОЯНИЯМ (только с переходами за период)
================================================================================
Период анализа: 2025-05-01 00:00:00 – 2025-06-01 00:00:00
Аварийных состояний с переходами за период: 2

================================================================================
Всего инцидентов: 5 (незавершённых: 1)
Распределение по приоритетам: high: 3, medium: 2
Средняя длительность завершённых инцидентов (минут): 12.3
...

--------------------------------------------------------------------------------
```

## 5. mchain_health_check

**Назначение**  
Быстрая диагностика состояния цепи Маркова. Возвращает статус (`OK`, `WARNING`, `CRITICAL`) с поясняющим сообщением. Предназначена для систем мониторинга (Zabbix, Prometheus и др.).

**Сигнатура**  
```sql
mchain_health_check()
RETURNS TABLE (status TEXT, message TEXT)
```

**Входные параметры**  
Нет.

**Возвращаемое значение**  
Таблица из одной строки с двумя колонками:

- `status` – `'OK'`, `'WARNING'` или `'CRITICAL'`.
- `message` – текстовое сообщение о проблеме или подтверждение штатной работы.

**Логика работы (проверки)**

| № | Проверка | Условие для WARNING | Условие для CRITICAL |
|---|----------|---------------------|----------------------|
| 1 | Достоверность прогнозов (рейтинг 0–5) | рейтинг 1–2 | рейтинг = 0 |
| 2 | Рост аварийных переходов (последние 7 дней vs предыдущие 7) – **аварийные определяются как** `correlation<0 AND os_trend=-1 AND wait_trend=1` | рост > 2× | рост > 3× |
| 3 | Забывание: давность последнего вызова `mchain_apply_forgetting` | `last_forget_time < now() - 2*interval_minute` | (только WARNING) |
| 4 | Активность: есть ли переходы за последние 10 минут | – | отсутствие переходов |
| 5 | Наличие данных в `markov_frequencies` | таблица пуста | (только WARNING) |

Статус повышается от `OK` → `WARNING` → `CRITICAL` по мере обнаружения проблем. Сообщения накапливаются через разделитель `'; '`.

**Примеры вызова и вывода**

```sql
SELECT * FROM mchain_health_check();
```

**Нормальная работа:**
```
 status | message
--------+---------------------------------------------------------
 OK     | Все проверки пройдены. Цепь Маркова работает штатно.
```

**Предупреждение:**
```
 status  | message
---------+---------------------------------------------------------
 WARNING | Низкая достоверность (2). Требуется больше данных.; Значительный рост аварийных переходов (0.5% -> 1.8%)
```

**Критическое состояние:**
```
 status   | message
----------+---------------------------------------------------------
 CRITICAL | Модель не обучена (рейтинг 0). Нет данных или менее 100 переходов.; Нет переходов за последние 10 минут (возможно, остановлен mchain_train_step).
```

## 6. mchain_state_transition_matrix_report

**Назначение**  
Формирует текстовую матрицу переходов между **укрупнёнными группами состояний**. Группировка выполняется по знаку корреляции (Neg/Zero/Pos) и тренду операционной скорости (down/stable/up), а также опционально – по тренду ожиданий. Источник данных – `markov_probabilities` (усреднение вероятностей по состояниям внутри группы).

**Сигнатура**  
```sql
mchain_state_transition_matrix_report(
    p_use_weighted BOOLEAN DEFAULT FALSE,
    p_include_wait_trend BOOLEAN DEFAULT FALSE
)
RETURNS TEXT
```

**Входные параметры**  
| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|-----------|
| `p_use_weighted` | BOOLEAN | FALSE | Если TRUE – вероятности усредняются с весами `frequency` из `markov_frequencies` (взвешенное среднее). Если FALSE – простое среднее арифметическое. |
| `p_include_wait_trend` | BOOLEAN | FALSE | Если TRUE – в группировку включается тренд ожиданий (wait_trend), что даёт 27 групп (3×3×3). Если FALSE – только корреляция и тренд OS (9 групп). |

**Возвращаемое значение**  
Многострочный текст (TEXT), содержащий:

- Заголовок с описанием параметров группировки и способа усреднения.
- Матрицу переходов: строки – исходные группы, столбцы – целевые группы, на пересечении – усреднённая вероятность перехода (округлённая до 3 знаков).
- Отдельный блок с переходами из **текущего состояния** (если оно определено) – распределение вероятностей по макрогруппам.

Если таблица `markov_probabilities` пуста или отсутствует `state_descriptions`, возвращается сообщение об ошибке.

**Логика работы**

1. Проверяет существование и заполненность необходимых таблиц.
2. Создаёт временную таблицу `matrix_agg`, где для каждой пары групп (`from_group`, `to_group`) вычисляется:
   - При `p_use_weighted = TRUE`:  
     `SUM(probability * frequency) / SUM(frequency)` по всем состояниям, входящим в группу.
   - При `p_use_weighted = FALSE`:  
     `AVG(probability)`.
3. Формирует заголовки групп (сокращённые обозначения, например `Neg_down` или `Neg_down_down`).
4. Выводит матрицу в текстовом виде с фиксированной шириной колонок.
5. Дополнительно определяет текущее состояние (через `mchain_get_current_state_id()`), находит его макрогруппу и выводит вероятности переходов из этой группы в другие.

**Пример вызова**

```sql
-- Матрица 9×9 (простое среднее)
SELECT mchain_state_transition_matrix_report();

-- Матрица 27×27 с взвешиванием по частоте
SELECT mchain_state_transition_matrix_report(p_use_weighted := TRUE, p_include_wait_trend := TRUE);
```

**Пример вывода (фрагмент для 9 групп)**

```
------------------------------------------------------------------------------------------------------------------------
МАТРИЦА ПЕРЕХОДОВ МЕЖДУ МАКРОГРУППАМИ СОСТОЯНИЙ
========================================================================================================================
Группировка: знак корреляции (Neg/Zero/Pos) + тренд OS (down/stable/up) = 9 групп
Способ усреднения: простое среднее

From \ To    Neg_down   Neg_stable Neg_up     Zero_down  Zero_stable Zero_up    Pos_down   Pos_stable Pos_up    
------------------------------------------------------------------------------------------------------------------------
Neg_down     0.423      0.312      0.112      0.089      0.045      0.012      0.004      0.002      0.001     
Neg_stable   0.221      0.401      0.189      0.087      0.062      0.023      0.008      0.005      0.004     
...
```

## Заключение

Представленные функции предоставляют полный набор инструментов для:

- **Оценки качества модели** – `mchain_reliability_report`.
- **Анализа аварийных переходов** – `mchain_incident_transitions_report` (общий) и `mchain_incident_state_detail_report` (детальный с привязкой к инцидентам).
- **Матричного анализа переходов** – `mchain_state_transition_matrix_report`.
- **Общего мониторинга** – `mchain_summary_report` (сводный) и `mchain_health_check` (для автоматических систем).

Используйте их регулярно для поддержания надёжности прогнозов и своевременной настройки параметров забывания.

> **Примечание по определению аварийных состояний:**  
> Разные функции могут использовать разные критерии (с учётом или без учёта `wait_trend`). Всегда уточняйте определение в описании конкретной функции. В коде это отражено явно.
