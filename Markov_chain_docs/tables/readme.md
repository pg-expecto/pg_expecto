## Общее описание

Данный набор таблиц является ядром реализации **цепи Маркова для прогнозирования аварийных инцидентов** на основе потоковых метрик производительности СУБД. Модель обучается онлайн: каждую минуту вызывается функция `mchain_train_step()`, которая фиксирует переход между состояниями системы. В среднем происходит **1 инцидент в день**, что учитывается при адаптивном забывании.

Таблицы спроектированы для:
- **Хранения частот переходов** и их вероятностей.
- **Ведения полного журнала переходов** для анализа и проверки достаточности данных.
- **Поддержки поглощающей матрицы** для многошагового прогнозирования риска.
- **Архивирования снимков модели** для отслеживания дрейфа.
- **Логирования работы адаптивного забывания** и ошибок.

Все таблицы создаются скриптом `markov_chain_tables.sql`. Скрипт идемпотентен (использует `DROP TABLE IF EXISTS` и `CREATE TABLE IF NOT EXISTS`).

## Конфигурация и управление

### `markov_config`
Хранит все настройки обучения, забывания, пороги и временные метки. Таблица содержит **одну строку** (синглтон).

| Колонка | Тип | По умолчанию | Описание |
|---------|-----|--------------|----------|
| `last_forget_time` | TIMESTAMPTZ | `now()` | Время последнего вызова забывания |
| `alpha` | REAL | 0.1 | Скорость забывания (если не адаптивный режим) |
| `interval_minute` | INT | 30 | Минимальный интервал между забываниями (минуты) |
| `forecast_log_retention_days` | SMALLINT | 21 | Срок хранения прогнозов |
| `transition_log_retention_days` | SMALLINT | 21 | Срок хранения переходов |
| `adaptive_forgetting_enabled` | BOOLEAN | `true` | Глобальный флаг забывания |
| `archive_retention_days` | SMALLINT | 21 | Срок хранения архивных матриц |
| `use_adaptive_alpha` | BOOLEAN | `true` | Включить динамический alpha |
| `base_alpha` | REAL | 0.1 | Базовый alpha (при частых инцидентах) |
| `min_alpha` | REAL | 0.01 | Минимальный alpha |
| `incident_half_life_days` | REAL | 7.0 | Период полураспада веса инцидента (дни) |
| `last_incident_time` | TIMESTAMPTZ | `NULL` | Время последнего аварийного перехода (обновляется триггером) |
| `apply_forgetting_log_retention_days` | INT | 21 | Срок хранения лога вызовов забывания |
| `last_snapshot_date` | DATE | `current_date` | Дата последнего снимка матрицы |
| `min_transitions_for_forgetting` | INT | 5000 | Минимальное число переходов для включения забывания |

**Особенности:**  
- Обновляется автоматически триггером при аварийном переходе (`last_incident_time`).  
- Используется всеми `mchain_*` функциями.

## Основные таблицы обучения

### `markov_chain`
Хранит **текущее и предыдущее состояния** системы. Всегда содержит одну строку (UNLOGGED для производительности).

| Колонка | Тип | Описание |
|---------|-----|----------|
| `prev_correlation` | REAL | Предыдущее значение корреляции |
| `prev_os_trend` | SMALLINT | Предыдущий тренд скорости |
| `prev_wait_trend` | SMALLINT | Предыдущий тренд ожиданий |
| `curr_correlation` | REAL | Текущее значение корреляции (NOT NULL) |
| `curr_os_trend` | SMALLINT | Текущий тренд скорости (NOT NULL) |
| `curr_wait_trend` | SMALLINT | Текущий тренд ожиданий (NOT NULL) |

**Использование:** обновляется каждую минуту в `mchain_train_step`.

### `transition_log`
Журнал всех переходов между состояниями (история наблюдений).

| Колонка | Тип | Описание |
|---------|-----|----------|
| `id` | BIGSERIAL | Первичный ключ |
| `ts` | TIMESTAMPTZ | Время перехода (по умолчанию `now()`) |
| `from_state` | SMALLINT | Исходное состояние |
| `to_state` | SMALLINT | Целевое состояние |

**Индексы:** `(ts)`, `(from_state)`, `(ts, from_state)`.  
**Использование:** каждая минута добавляется одна запись. Используется для проверки достаточности обучения и пересчёта эталонов.

### `markov_frequencies`
Накопление сырых частот переходов (основа для вероятностей). Поддерживает дробные значения после забывания.

| Колонка | Тип | Описание |
|---------|-----|----------|
| `from_state` | SMALLINT | Исходное состояние |
| `to_state` | SMALLINT | Целевое состояние |
| `frequency` | REAL | Накопленная частота (не обязательно целое) |

**Primary Key:** `(from_state, to_state)`.  
**Использование:** обновляется при каждом переходе (`frequency++`) и при забывании (умножение на `1-alpha`). Строки с `frequency < 1e-6` удаляются.

## Таблицы вероятностей и поглощения

### `markov_probabilities`
Матрица условных вероятностей переходов, рассчитанная из `markov_frequencies`.

| Колонка | Тип | Описание |
|---------|-----|----------|
| `from_state` | SMALLINT | Исходное состояние |
| `to_state` | SMALLINT | Целевое состояние |
| `probability` | REAL | Условная вероятность P(to_state \| from_state) |

**PK:** `(from_state, to_state)`.  
**Использование:** пересчитывается функцией `update_markov_probabilities()` при каждом забывании или вручную.

### `markov_absorbing`
Поглощающая матрица для многошагового прогноза риска. Аварийные состояния превращены в поглощающие (петля с вероятностью 1.0).

| Колонка | Тип | Описание |
|---------|-----|----------|
| `from_state` | SMALLINT | Исходное состояние |
| `to_state` | SMALLINT | Целевое состояние |
| `probability` | REAL | Вероятность перехода |

**PK:** `(from_state, to_state)`.  
**Использование:** строится функцией `rebuild_markov_absorbing()` из `markov_probabilities`. Используется функциями прогноза `mchain_predict_risk_k`.

## Справочники

### `state_descriptions`
Фиксированный справочник всех 189 возможных состояний. Заполняется один раз функцией `fill_state_descriptions()`.

| Колонка | Тип | Описание |
|---------|-----|----------|
| `state_id` | SMALLINT | Первичный ключ (0..188) |
| `correlation` | REAL | Коэффициент корреляции (округлённый до 0.1) |
| `os_trend` | SMALLINT | Тренд операционной скорости (-1,0,1) |
| `wait_trend` | SMALLINT | Тренд времени ожидания (-1,0,1) |

**Использование:** идентификация аварийных состояний (`correlation < 0 AND os_trend = -1`), получение `state_id` по метрикам.

## Архивы и снимки

### `markov_probabilities_archive`
Архив версий матрицы вероятностей с привязкой к дате обучения.

| Колонка | Тип | Описание |
|---------|-----|----------|
| `train_date` | DATE | Дата фиксации модели |
| `from_state` | SMALLINT | Исходное состояние |
| `to_state` | SMALLINT | Целевое состояние |
| `probability` | REAL | Вероятность перехода |

**PK:** `(train_date, from_state, to_state)`.  
**Индекс:** `(train_date, from_state)`.  
**Использование:** заполняется функцией `archive_markov_probabilities()`, очищается по `archive_retention_days`.

### `markov_probabilities_prev_week`
Снимок матрицы вероятностей на момент последнего еженедельного архивирования (пятница, 19:05).

| Колонка | Тип | Описание |
|---------|-----|----------|
| `from_state` | SMALLINT | Исходное состояние |
| `to_state` | SMALLINT | Целевое состояние |
| `probability` | REAL | Вероятность перехода |

**PK:** `(from_state, to_state)`.  
**Использование:** обновляется функцией `mchain_snapshot_prev_week()` для сравнения стабильности.

### `state_baseline`
Эталонное распределение состояний по часам и дням недели (для KL-дивергенции и обнаружения дрейфа).

| Колонка | Тип | Описание |
|---------|-----|----------|
| `hour_of_day` | INT | Час (0..23) |
| `dow` | INT | День недели (1..7, где 1 = понедельник) |
| `state_id` | SMALLINT | Состояние |
| `probability` | REAL | Ожидаемая вероятность |
| `last_updated` | TIMESTAMPTZ | Время последнего обновления |

**Использование:** обновляется функцией `mchain_update_baseline()` (cron). Может использоваться для мониторинга дрейфа, но в текущей версии не задействована в прогнозах.

### `operational_speed_stats`
Статистика операционной скорости по часам (средняя и стандартное отклонение).

| Колонка | Тип | Описание |
|---------|-----|----------|
| `hour_of_day` | INT | Час (0..23) |
| `avg_speed` | REAL | Средняя операционная скорость |
| `stddev_speed` | REAL | Стандартное отклонение |
| `last_updated` | TIMESTAMPTZ | Время последнего обновления |

**Использование:** обновляется функцией `mchain_refresh_os_stats()` (cron). Может использоваться для аномалий, но не является критической для цепи.

## Журналы и логи

### `apply_forgetting_log`
Журнал всех вызовов функции забывания (успешных и пропущенных).

| Колонка | Тип | Описание |
|---------|-----|----------|
| `id` | BIGSERIAL | PK |
| `ts` | TIMESTAMPTZ | Время вызова |
| `effective_alpha` | REAL | Фактически применённый коэффициент |
| `adaptive_used` | BOOLEAN | Был ли включён адаптивный режим |
| `days_since_incident` | REAL | Число дней с последнего инцидента |
| `alpha_override` | REAL | Если передан параметр `alpha_override` |
| `details` | TEXT | Детали расчёта `alpha` |

**Использование:** заполняется в `mchain_apply_forgetting()`. Очищается по `apply_forgetting_log_retention_days`.

### `mchain_error_log`
Журнал ошибок, возникающих при работе `mchain_*` функций.

| Колонка | Тип | Описание |
|---------|-----|----------|
| `id` | BIGSERIAL | PK |
| `ts` | TIMESTAMPTZ | Время ошибки |
| `function_name` | TEXT | Имя функции, где произошла ошибка |
| `error_message` | TEXT | Текст ошибки (SQLERRM) |
| `error_detail` | TEXT | Детали (SQLSTATE и пр.) |
| `error_hint` | TEXT | Подсказка |
| `context` | JSONB | Дополнительный контекст (параметры) |

**Использование:** заполняется функцией `mchain_log_error()`.

### `forecast_log`
Журнал прогнозов риска и фактических исходов (в текущей версии **не используется**, таблица создана для совместимости).

**Примечание:** в `markov_chain_tables.sql` эта таблица закомментирована, так как прогнозы не записываются автоматически.

## Служебные таблицы (не используемые в mchain_*)

Следующие таблицы упоминаются в старых версиях, но **не используются** функциями `mchain_*`. Они оставлены для обратной совместимости или не создаются вообще:

- `forget_log` – журнал форсированных забываний (не используется, очистка через `mchain_clean_forget_log` всё же есть).
- `infrastructure_events` – регистрация внешних событий.
- `check_state` – состояние проверок.

В актуальной реализации они либо закомментированы, либо не имеют триггеров.

## Связи между таблицами

Ниже представлена ER-диаграмма (текстовое описание):

```
markov_config (1 строка) <---- триггер обновляет last_incident_time
       |
       | interval_minute, alpha и др.
       v
mchain_train_step() ---> transition_log ---> markov_frequencies
                              |                      |
                              | (триггер)            v
                              +------------> update_markov_probabilities()
                                                    |
                                                    v
                                            markov_probabilities
                                                    |
                                                    v
                                            rebuild_markov_absorbing()
                                                    |
                                                    v
                                            markov_absorbing
                                                    |
                                                    v
                                            mchain_predict_risk_k()

state_descriptions <--- используется get_state_id() и для определения аварийных состояний

markov_chain --- хранит текущее/предыдущее состояние

markov_probabilities_archive <--- archive_markov_probabilities()
markov_probabilities_prev_week <--- mchain_snapshot_prev_week()

state_baseline <--- mchain_update_baseline()
operational_speed_stats <--- mchain_refresh_os_stats()

apply_forgetting_log <--- mchain_apply_forgetting()
mchain_error_log <--- mchain_log_error()
```

**Внешние зависимости:**  
Таблица `cluster_stat_median` (не входит в этот набор) предоставляет метрики для `get_current_os_waiting_correlation_for_markov_chain`.

## Резюме

Набор таблиц обеспечивает полный цикл онлайн-обучения цепи Маркова с адаптивным забыванием и прогнозированием риска. Ключевые особенности:
- **Минутная гранулярность** обучения.
- **Автоматическое обновление** вероятностей и поглощающей матрицы.
- **Гибкая настройка** забывания через `markov_config`.
- **Полная аудируемость** (логи переходов, вызовов забывания, ошибок).
- **Архивация** снимков модели для анализа дрейфа.

Для работы достаточно создать таблицы, настроить источник метрик `cluster_stat_median` и запустить минутный вызов `mchain_train_step()`.
