# Описание таблиц системы марковской цепи для прогнозирования рисков

Система реализует марковскую модель для оценки вероятности перехода в аварийное состояние (отрицательная корреляция + падение операционной скорости). В основе лежат частоты переходов между состояниями, которые обновляются в реальном времени, подвергаются забыванию и архивируются. Ниже приведено описание всех таблиц, используемых в системе.

---

## 1. Конфигурация и управление

### `markov_config`
Таблица глобальных настроек модели: параметры забывания, пороги срабатывания, сроки хранения данных и флаги адаптивного режима.

| Столбец | Тип | Описание |
|---------|-----|-----------|
| `last_forget_time` | TIMESTAMPTZ | Время последнего применения планового забывания |
| `alpha` | REAL | Скорость забывания (базовый коэффициент) |
| `interval_minute` | INT | Интервал планового забывания в минутах |
| `forecast_log_retention_days` | SMALLINT | Глубина хранения записей в `forecast_log` (дней) |
| `transition_log_retention_days` | SMALLINT | Глубина хранения записей в `transition_log` (дней) |
| `kl_threshold` | REAL | Порог KL-дивергенции для форсированного забывания |
| `chi2_threshold` | REAL | Порог χ² для форсированного забывания (для 188 степеней свободы) |
| `os_dev_threshold` | REAL | Порог отклонения операционной скорости (например, 0.3 = 30%) |
| `brier_threshold` | REAL | Порог Brier Score (0.25) |
| `forget_alpha_max` | REAL | Максимально допустимый коэффициент забывания при форсировании |
| `confirmation_cycles` | SMALLINT | Сколько последовательных проверок должно подтвердить признак для срабатывания |
| `adaptive_forgetting_enabled` | BOOLEAN | Разрешает ли функция `check_and_forget` выполнять забывание |
| `archive_retention_days` | SMALLINT | Срок хранения архивных снимков матрицы (дней) |
| `check_state_retention_days` | SMALLINT | Срок хранения истории проверок `check_state` (дней) |
| `forget_log_retention_days` | SMALLINT | Срок хранения журнала форсированных забываний (дней) |
| `brier_min_observations` | SMALLINT | Минимальное количество прогнозов за 2 часа для расчёта Brier Score |
| `use_adaptive_alpha` | BOOLEAN | Включено ли адаптивное изменение alpha в зависимости от времени с последнего инцидента |
| `base_alpha` | REAL | Базовое значение alpha (при частых инцидентах) |
| `min_alpha` | REAL | Минимальное значение alpha (при очень редких инцидентах) |
| `incident_half_life_days` | REAL | Период полураспада веса инцидента (дни) |
| `last_incident_time` | TIMESTAMPTZ | Время последнего перехода в аварийное состояние |
| `apply_forgetting_log_retention_days` | INT | Срок хранения записей в `apply_forgetting_log` (дней) |
| `last_snapshot_date` | DATE | Дата последнего снимка матрицы (используется как model_train_date) |

Первичный ключ отсутствует – таблица содержит единственную строку (настраивается однократно).

## 2. Основные данные цепи Маркова

### `markov_frequencies`
Хранит частоты переходов между состояниями. Из этих частот затем вычисляются вероятности.

| Столбец | Тип | Описание |
|---------|-----|-----------|
| `from_state` | SMALLINT | Исходное состояние (0..188) |
| `to_state` | SMALLINT | Целевое состояние (0..188) |
| `frequency` | REAL | Количество наблюдённых переходов (или взвешенная частота после забывания) |

Первичный ключ: `(from_state, to_state)`.

### `markov_probabilities`
Матрица вероятностей перехода, полученная нормировкой `markov_frequencies` по строкам.

| Столбец | Тип | Описание |
|---------|-----|-----------|
| `from_state` | SMALLINT | Исходное состояние |
| `to_state` | SMALLINT | Целевое состояние |
| `probability` | REAL | Вероятность перехода `P(from_state → to_state)` |

Первичный ключ: `(from_state, to_state)`.

### `markov_absorbing`
Поглощающая матрица, в которой все аварийные состояния сделаны поглощающими (вероятность остаться в себе = 1). Используется для расчёта риска за K шагов.

| Столбец | Тип | Описание |
|---------|-----|-----------|
| `from_state` | SMALLINT | Исходное состояние |
| `to_state` | SMALLINT | Целевое состояние |
| `probability` | REAL | Вероятность перехода (1.0 для аварийных петель, иначе из `markov_probabilities`) |

Первичный ключ: `(from_state, to_state)`.

### `state_descriptions`
Справочник, сопоставляющий числовому `state_id` трёхмерную метку: корреляцию, тренд операционной скорости и тренд ожиданий.

| Столбец | Тип | Описание |
|---------|-----|-----------|
| `state_id` | SMALLINT | Уникальный идентификатор состояния (0..188) |
| `correlation` | REAL | Округлённое значение коэффициента корреляции (шаг 0.1, от -1.0 до +1.0) |
| `os_trend` | SMALLINT | Тренд операционной скорости: -1 (падение), 0 (стабильно), +1 (рост) |
| `wait_trend` | SMALLINT | Тренд ожиданий: -1, 0, +1 |

Первичный ключ: `(state_id)`.

### `markov_chain`
Хранит текущее и предыдущее состояние системы в реальном времени, используемое в цикле обучения.

| Столбец | Тип | Описание |
|---------|-----|-----------|
| `prev_correlation` | REAL | Предыдущее значение корреляции |
| `prev_os_trend` | SMALLINT | Предыдущий тренд операционной скорости |
| `prev_wait_trend` | SMALLINT | Предыдущий тренд ожиданий |
| `curr_correlation` | REAL | Текущее значение корреляции (NOT NULL) |
| `curr_os_trend` | SMALLINT | Текущий тренд операционной скорости (NOT NULL) |
| `curr_wait_trend` | SMALLINT | Текущий тренд ожиданий (NOT NULL) |

Таблица содержит одну строку. Первичного ключа нет.

## 3. Журналы событий

### `transition_log`
Хронология всех переходов между состояниями (каждое вычисленное состояние, получаемое из метрик каждую минуту).

| Столбец | Тип | Описание |
|---------|-----|-----------|
| `id` | BIGSERIAL | Первичный ключ |
| `ts` | TIMESTAMPTZ | Время наблюдения (по умолчанию now()) |
| `from_state` | SMALLINT | Исходное состояние |
| `to_state` | SMALLINT | Целевое состояние |

Индексы: `idx_transition_log_ts`, `idx_transition_log_from`, `idx_transition_log_ts_from`.

### `forecast_log`
Журнал прогнозов (одношаговая вероятность аварии) и фактических исходов. Используется для оценки точности модели (Brier Score).

| Столбец | Тип | Описание |
|---------|-----|-----------|
| `id` | BIGSERIAL | Первичный ключ |
| `ts` | TIMESTAMPTZ | Время, когда стал известен фактический исход |
| `model_train_date` | DATE | Дата, до которой обучена модель (идентификатор версии) |
| `predicted_risk` | REAL | Предсказанная вероятность аварии на следующем шаге |
| `actual_risk` | SMALLINT | 1, если переход был в аварийное состояние, иначе 0 |
| `from_state` | SMALLINT | Исходное состояние (опционально) |
| `to_state` | SMALLINT | Фактическое состояние (опционально) |

Индексы: `idx_forecast_log_model_date`, `idx_forecast_log_ts`, `idx_forecast_log_ts_2h` (частичный), `idx_forecast_log_ts_predicted`.

### `forget_log`
Журнал событий форсированного забывания, инициированных функцией `check_and_forget`.

| Столбец | Тип | Описание |
|---------|-----|-----------|
| `id` | BIGSERIAL | Первичный ключ |
| `ts` | TIMESTAMPTZ | Время события |
| `alpha` | REAL | Применённый коэффициент забывания |
| `triggered_by` | TEXT[] | Массив признаков, вызвавших забывание (KL, OS, Brier, Infra, Diurnal) |
| `kl_div` | REAL | Значение KL-дивергенции в момент проверки |
| `chi2_val` | REAL | Значение χ² |
| `brier_score` | REAL | Brier Score за последние 2 часа |
| `os_deviation` | REAL | Отклонение операционной скорости |
| `details` | TEXT | Дополнительная информация |

### `apply_forgetting_log`
Журнал всех вызовов `apply_forgetting` (как плановых, так и форсированных).

| Столбец | Тип | Описание |
|---------|-----|-----------|
| `id` | BIGSERIAL | Первичный ключ |
| `ts` | TIMESTAMPTZ | Время вызова |
| `effective_alpha` | REAL | Фактически применённый коэффициент забывания |
| `adaptive_used` | BOOLEAN | Использовалось ли адаптивное забывание (из `use_adaptive_alpha`) |
| `days_since_incident` | REAL | Количество дней с последнего инцидента (NULL, если нет) |
| `alpha_override` | REAL | Значение параметра `alpha_override` (если был передан) |
| `details` | TEXT | Детали вычисления alpha |

### `check_state`
Хранит результаты каждой проверки `check_and_forget` для механизма подтверждения (контроль последовательных срабатываний).

| Столбец | Тип | Описание |
|---------|-----|-----------|
| `check_time` | TIMESTAMPTZ | Время проверки (первичный ключ) |
| `kl_flag` | BOOLEAN | Превышен порог KL-дивергенции? |
| `chi2_flag` | BOOLEAN | Превышен порог χ²? |
| `os_flag` | BOOLEAN | Превышен порог отклонения скорости? |
| `brier_flag` | BOOLEAN | Превышен порог Brier Score? |
| `infra_flag` | BOOLEAN | Было ли событие инфраструктуры за последний час? |
| `diurnal_flag` | BOOLEAN | KL-дивергенция текущего часа с эталоном > 0.2? |

### `infrastructure_events`
Регистрация внешних событий (деплой, изменение конфигурации, сбой и т.п.), которые могут повлиять на поведение системы и инициировать забывание.

| Столбец | Тип | Описание |
|---------|-----|-----------|
| `id` | BIGSERIAL | Первичный ключ |
| `event_time` | TIMESTAMPTZ | Время события (по умолчанию now()) |
| `event_type` | TEXT | Тип события: 'deploy', 'config_change', 'failover', 'manual' |
| `description` | TEXT | Дополнительное описание |
| `processed` | BOOLEAN | Флаг, обработано ли событие (сбрасывается при применении забывания) |

## 4. Снимки и архивы

### `markov_probabilities_prev_week`
Еженедельный снимок матрицы вероятностей (создаётся по cron в пятницу в 19:05). Используется для сравнения стабильности модели (критерий 2 достаточности обучения).

| Столбец | Тип | Описание |
|---------|-----|-----------|
| `from_state` | SMALLINT | Исходное состояние |
| `to_state` | SMALLINT | Целевое состояние |
| `probability` | REAL | Вероятность перехода на момент снимка |

Первичный ключ: `(from_state, to_state)`.

### `markov_probabilities_archive`
Архив исторических снимков матрицы вероятностей с привязкой к дате обучения. Используется для ретроспективного прогнозирования и оценки достаточности обучения.

| Столбец | Тип | Описание |
|---------|-----|-----------|
| `train_date` | DATE | Дата, на которую зафиксирована модель (последний день обучающих данных) |
| `from_state` | SMALLINT | Исходное состояние |
| `to_state` | SMALLINT | Целевое состояние |
| `probability` | REAL | Вероятность перехода |

Первичный ключ: `(train_date, from_state, to_state)`.  
Индекс: `idx_archive_date_from` для быстрого прогноза по модели.

## 5. Статистика и эталоны

### `operational_speed_stats`
Среднее и стандартное отклонение операционной скорости по часам дня (рассчитывается за последние 20 дней). Используется для вычисления `get_os_deviation()`.

| Столбец | Тип | Описание |
|---------|-----|-----------|
| `hour_of_day` | SMALLINT | Час (0..23) – первичный ключ |
| `avg_speed` | REAL | Средняя операционная скорость в этот час |
| `stddev_speed` | REAL | Стандартное отклонение |
| `last_updated` | TIMESTAMPTZ | Время последнего обновления |

### `state_baseline`
Эталонное распределение состояний для каждого часа и дня недели (рассчитывается за последние 7 дней). Используется для расчёта KL-дивергенции и χ² в функции `check_and_forget`.

| Столбец | Тип | Описание |
|---------|-----|-----------|
| `hour_of_day` | SMALLINT | Час (0..23) |
| `dow` | SMALLINT | День недели (1 = понедельник … 7 = воскресенье) |
| `state_id` | SMALLINT | Идентификатор состояния |
| `probability` | REAL | Вероятность застать систему в этом состоянии в данном слоте |
| `last_updated` | TIMESTAMPTZ | Время последнего обновления |

Первичный ключ: `(hour_of_day, dow, state_id)`.

---

> **Примечание:** Все перечисленные таблицы создаются скриптом `markov_chain_tables.sql` (версия 10.0.3). Система предполагает наличие дополнительной таблицы `cluster_stat_median` (не входит в данный набор), которая поставляется отдельно и содержит медианные значения операционной скорости и ожиданий за минуту.
