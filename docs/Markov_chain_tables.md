# Описание таблиц реализации цепи Маркова для анализа производительности СУБД PostgreSQL

## Общее назначение

Представленные таблицы реализуют **цепь Маркова** для моделирования и прогнозирования состояний производительности PostgreSQL.  
Система кодирует состояние производительности тремя параметрами:

- **correlation** — округлённый коэффициент корреляции (вероятно, между метриками)
- **os_trend** — тренд операционной скорости (`-1`, `0`, `+1`)
- **wait_trend** — тренд ожиданий (задержек) (`-1`, `0`, `+1`)

Комбинация этих трёх дискретизированных признаков образует **конечное состояние** (всего 189 возможных состояний: 21 значение корреляции × 3 тренда × 3 тренда).  
Модель позволяет:

- Обучать матрицу переходных вероятностей на основе наблюдаемых переходов
- Автоматически «забывать» устаревшие данные (адаптация к изменению нагрузки)
- Прогнозировать риск аварийного состояния
- Оценивать точность прогнозов и архивировать версии моделей

---

## 1. `markov_frequencies` — основная таблица переходных частот

Хранит **сырые частоты переходов** между состояниями. Каждое обновление (новый переход) увеличивает частоту на единицу.  
Используется как основа для расчёта вероятностей.

| Столбец      | Тип        | Описание                                 |
|--------------|------------|------------------------------------------|
| `from_state` | `SMALLINT` | Исходное состояние (0..188)             |
| `to_state`   | `SMALLINT` | Целевое состояние (0..188)              |
| `frequency`  | `REAL`     | Накопленная частота перехода (≥0)       |

**Первичный ключ:** `(from_state, to_state)`

**Особенности:**  
- Построчное хранение позволяет атомарно обновлять одну ячейку (`INSERT ... ON CONFLICT DO UPDATE`)  
- Поддерживает операцию «забывания»: `UPDATE markov_frequencies SET frequency = frequency * (1 - alpha)`  
- Быстрый выбор всех переходов из заданного состояния по `from_state`

---

## 2. `transition_log` — журнал переходов

Фиксирует **каждый наблюдаемый переход** во времени. Служит исходными данными для обучения модели и аудита.

| Столбец      | Тип           | Описание                                     |
|--------------|---------------|----------------------------------------------|
| `id`         | `BIGSERIAL`   | Уникальный идентификатор записи             |
| `ts`         | `TIMESTAMPTZ` | Момент времени перехода (по умолчанию `now()`) |
| `from_state` | `SMALLINT`    | Исходное состояние                          |
| `to_state`   | `SMALLINT`    | Целевое состояние                           |

**Первичный ключ:** `id`  
**Индексы:**  
- `idx_transition_log_ts` — для выборок по времени  
- `idx_transition_log_from` — для выборок по исходному состоянию

---

## 3. `markov_probabilities` — матрица вероятностей переходов

Содержит **рассчитанные вероятности** переходов (нормированные по строкам).  
Обновляется периодически (например, после каждого «забывания» или по расписанию) на основе данных из `markov_frequencies`.

| Столбец        | Тип        | Описание                                     |
|----------------|------------|----------------------------------------------|
| `from_state`   | `SMALLINT` | Исходное состояние                          |
| `to_state`     | `SMALLINT` | Целевое состояние                           |
| `probability`  | `REAL`     | Вероятность перехода `P(to_state | from_state)` |

**Первичный ключ:** `(from_state, to_state)`  
**Примечание:** Для каждого `from_state` сумма `probability` по всем `to_state` должна равняться 1.0 (с точностью `REAL`).

---

## 4. `markov_probabilities_prev_week` — снимок матрицы за предыдущую неделю

Хранит копию матрицы вероятностей, полученную **неделю назад**.  
Используется для сравнения изменений модели или для отката к предыдущей стабильной версии.

| Столбец        | Тип        | Описание                                     |
|----------------|------------|----------------------------------------------|
| `from_state`   | `SMALLINT` | Исходное состояние                          |
| `to_state`     | `SMALLINT` | Целевое состояние                           |
| `probability`  | `REAL`     | Вероятность перехода (недельной давности)  |

**Первичный ключ:** `(from_state, to_state)`

---

## 5. `markov_absorbing` — поглощающая матрица

Таблица для **поглощающих состояний** (например, состояния аварийного отказа).  
Может использоваться для расчёта времени до поглощения или вероятности аварии.

| Столбец        | Тип        | Описание                                     |
|----------------|------------|----------------------------------------------|
| `from_state`   | `SMALLINT` | Исходное состояние                          |
| `to_state`     | `SMALLINT` | Целевое состояние (обычно аварийное)        |
| `probability`  | `REAL`     | Вероятность перехода                         |

**Первичный ключ:** `(from_state, to_state)`

---

## 6. `state_descriptions` — справочник состояний

Расшифровывает числовой идентификатор состояния (0..188) в три составляющие.  
Необходим для интерпретации результатов моделирования.

| Столбец         | Тип        | Описание                                                         |
|-----------------|------------|------------------------------------------------------------------|
| `state_id`      | `SMALLINT` | Идентификатор состояния (первичный ключ)                        |
| `correlation`   | `REAL`     | Округлённое значение коэффициента корреляции                    |
| `os_trend`      | `SMALLINT` | Тренд операционной скорости: `-1` (падение), `0` (стабильно), `1` (рост) |
| `wait_trend`    | `SMALLINT` | Тренд ожиданий: `-1` (уменьшение), `0` (стабильно), `1` (увеличение) |

---

## 7. `markov_chain` — таблица для обучения (нежурналируемая)

Служит для **промежуточного обучения** модели. Хранит сырые значения признаков (не закодированные в состояние) и позволяет гибко строить переходы.

| Столбец            | Тип        | Описание                                                      |
|--------------------|------------|---------------------------------------------------------------|
| `prev_correlation` | `REAL`     | Предыдущее значение корреляции (может быть `NULL`)            |
| `prev_os_trend`    | `SMALLINT` | Предыдущий тренд операционной скорости                        |
| `prev_wait_trend`  | `SMALLINT` | Предыдущий тренд ожиданий                                     |
| `curr_correlation` | `REAL`     | Текущее значение корреляции (обязательно)                    |
| `curr_os_trend`    | `SMALLINT` | Текущий тренд операционной скорости (обязательно)            |
| `curr_wait_trend`  | `SMALLINT` | Текущий тренд ожиданий (обязательно)                         |

**Тип таблицы:** `UNLOGGED` — для повышения производительности при интенсивной вставке (данные не пишутся в WAL).  
**Назначение:** Используется как временное хранилище перед преобразованием в закодированные состояния и занесением в `transition_log`.

---

## 8. `forecast_log` — журнал прогнозов

Оценивает **качество прогнозов модели**. Содержит предсказанные вероятности риска и фактический исход.

| Столбец            | Тип           | Описание                                                              |
|--------------------|---------------|-----------------------------------------------------------------------|
| `id`               | `BIGSERIAL`   | Уникальный идентификатор                                             |
| `ts`               | `TIMESTAMPTZ` | Момент, когда стал известен фактический исход                        |
| `model_train_date` | `DATE`        | Дата, до которой обучена модель (идентификатор версии)               |
| `predicted_risk`   | `REAL`        | Предсказанная вероятность аварии на один шаг вперёд (0..1)           |
| `actual_risk`      | `SMALLINT`    | Факт: `1` — если переход был в аварийное состояние, иначе `0`        |
| `from_state`       | `SMALLINT`    | Исходное состояние (опционально, для детального анализа)             |
| `to_state`         | `SMALLINT`    | Фактическое целевое состояние (опционально)                          |

**Первичный ключ:** `id`  
**Индексы:**  
- `idx_forecast_log_model_date` — выборка прогнозов по версии модели  
- `idx_forecast_log_ts` — выборка по времени  

**Применение:** Позволяет вычислять метрики точности (AUC, Brier score, калибровку) для разных версий моделей.

---

## 9. `markov_config` — конфигурация забывания

Управляет механизмом **адаптивного забывания** устаревших данных.  
Периодически частоты умножаются на `(1 - alpha)`, что экспоненциально уменьшает вклад старых наблюдений.

| Столбец           | Тип           | Описание                                                         |
|-------------------|---------------|------------------------------------------------------------------|
| `last_forget_time`| `TIMESTAMPTZ` | Время последнего применения операции забывания                 |
| `alpha`           | `REAL`        | Коэффициент забывания (0..1). Рекомендуется `0.01`              |
| `interval_hours`  | `INT`         | Интервал между применениями забывания в часах (по умолчанию `1`) |

**Строка по умолчанию:** вставляется при создании таблицы.  
**Механизм:**  
1. Проверяется, прошло ли `interval_hours` с момента `last_forget_time`  
2. Если да — выполняется `UPDATE markov_frequencies SET frequency = frequency * (1 - alpha)`  
3. Обновляется `last_forget_time = now()`

---

## 10. `markov_probabilities_archive` — архив снимков матрицы вероятностей

Хранит **исторические версии** матрицы переходных вероятностей.  
Каждая версия привязана к дате обучения (`train_date`), что позволяет:

- Использовать модель «на дату» для ретроспективного прогнозирования
- Сравнивать эволюцию распределений переходов во времени
- Восстанавливать модель для повторного анализа

| Столбец        | Тип        | Описание                                                      |
|----------------|------------|---------------------------------------------------------------|
| `train_date`   | `DATE`     | Дата, на которую зафиксирована модель (последний день данных) |
| `from_state`   | `SMALLINT` | Исходное состояние                                           |
| `to_state`     | `SMALLINT` | Целевое состояние                                            |
| `probability`  | `REAL`     | Вероятность перехода `P(to_state | from_state)`              |

**Первичный ключ:** `(train_date, from_state, to_state)`  
**Индекс:** `idx_archive_date_from` — для быстрого получения всех переходов из состояния `from_state` для заданной даты `train_date`.

---

## Взаимосвязи таблиц (упрощённая схема)

```
transition_log          →  markov_frequencies  →  markov_probabilities
       │                              │                     │
       │                              │                     ├→ markov_probabilities_prev_week
       │                              │                     │
       │                              └→ markov_absorbing   │
       │                                                    │
       └→ forecast_log (прогнозы) ←─────────────────────────┘
       
markov_chain (временное обучение) → переходы в transition_log

state_descriptions — справочник для кодирования/декодирования

markov_config — управляет забыванием частот

markov_probabilities_archive — периодический снимок markov_probabilities
```

---

## Типовые операции над таблицами

| Операция                           | Задействованные таблицы                                   |
|------------------------------------|-----------------------------------------------------------|
| Добавление нового перехода         | `transition_log`, `markov_frequencies` (обновление +1)   |
| Расчёт вероятностей                | `markov_frequencies` → `markov_probabilities` (нормировка)|
| Адаптивное забывание               | `markov_config`, `markov_frequencies` (умножение частот)  |
| Прогноз риска на 1 шаг             | `markov_probabilities` + `state_descriptions`            |
| Оценка точности прогноза           | `forecast_log`                                            |
| Архивирование еженедельной модели  | `markov_probabilities` → `markov_probabilities_archive`  |
| Откат к предыдущей стабильной модели| `markov_probabilities_prev_week` → `markov_probabilities`|

---

## Замечания по производительности

- Все таблицы имеют минимально необходимые индексы и компактные типы (`SMALLINT`, `REAL`, `DATE`).
- `markov_chain` сделана `UNLOGGED` для высокой скорости вставки при обучении.
- Построчное хранение частот и вероятностей обеспечивает:
  - Быстрые точечные обновления (`ON CONFLICT`)
  - Эффективные массовые операции (`UPDATE` для забывания)
  - Удобную выборку переходов из одного состояния (`WHERE from_state = ?`)

Общий объём данных при 189 состояниях составляет:
- `markov_frequencies`: ~35 000 строк × ~35 байт ≈ 1.2 МБ
- Аналогично для `markov_probabilities`, `archive` и др. — общий размер остаётся в пределах десятков мегабайт, что легко обрабатывается PostgreSQL даже на ограниченных ресурсах.
