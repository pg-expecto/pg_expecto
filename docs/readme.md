# 📚 Документация модуля цепей Маркова (v.10.0)

Этот набор документов описывает реализацию и применение **Цепей Маркова** для прогнозирования производительности PostgreSQL в рамках проекта `pg_expecto`. 

Модуль позволяет моделировать переходы СУБД между состояниями, оценивать риски сбоев и обеспечивать адаптивное обучение на основе реальных данных.

## 🗂️ Содержание

*   **`Markov_chain_tables.md`** — описание структуры таблиц (`markov_frequencies`, `transition_log`, `markov_probabilities`), которые служат основой для хранения частот переходов и вероятностной матрицы.
*   **`Markov_chain_functions.md`** — подробная справка по всем хранимым функциям. Включает утилиты для кодирования состояний (`get_state_id`), учёта переходов (`log_transition_and_update`), прогнозирования рисков и т.д.
*   **`markov_chain_training.md`** — описывает основную процедуру обучения. Функция `markov_chain_training()` собирает метрики, обновляет частоты переходов и выполняет экспоненциальное «забывание» данных по расписанию в рабочие часы.
*   **`evaluate_training_sufficiency.md`** — руководство по функции `evaluate_training_sufficiency()`, которая оценивает качество обучения модели по четырём критериям (статистическая обеспеченность, стабильность вероятностей, Brier Score и стационарность распределения).

## 🚀 Назначение

Документация предназначена для администраторов баз данных и разработчиков, которые хотят внедрить предиктивный анализ производительности PostgreSQL с использованием статистических методов.
