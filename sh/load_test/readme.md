# Нагрузочное тестирование PostgreSQL (pg_expecto)

Система автоматического нагрузочного тестирования PostgreSQL с поддержкой смешанной нагрузки (OLTP/OLAP), плавным наращиванием числа клиентов и сохранением метрик. Скрипты написаны на Shell и SQL, используют `pgbench` в качестве генератора нагрузки.

---

## 📁 Состав

| Файл | Назначение |
|------|------------|
| `param.conf` | Конфигурация теста (диапазон нагрузки, веса сценариев, тип БД и др.). |
| `load_test_start.sh` | Инициализация теста: создание/подготовка БД, создание функций, стартовые настройки. |
| `load_test.sh` | Основной скрипт, запускающий одну итерацию теста (вызывается по расписанию). |
| `load_test_stop.sh` | Принудительная остановка теста и очистка. |
| `before_start.sql` | SQL-скрипт для подготовки тестовой БД (добавление столбца, отключение FK). |
| `get_conf_param.sh` | Чтение параметров из `param.conf`. |
| `get_vm_values.sh` | Сбор параметров ядра (vm.*) для сохранения. |
| `scenario[1-3].oltp.sql` | Функции сценариев для OLTP-нагрузки. |
| `scenario[1-3].olap.sql` | Функции сценариев для OLAP-нагрузки. |
| `do_scenario[1-3].sql` | Автоматически генерируемые скрипты вызова функций сценариев. |

---

## ⚙️ Требования

- **PostgreSQL** (версия 10+, рекомендуется 12+)
- **pgbench** (входит в состав PostgreSQL)
- **Bash** (для запуска скриптов)
- **Права на создание/удаление БД** (если используется режим `default`)
- **Права на чтение параметров ядра** (для `sysctl`)

## 🛠 Установка

1. Разместите все файлы в одном каталоге (например, `/postgres/pg_expecto/load_test/`).
2. Убедитесь, что скрипты имеют права на выполнение:
   ```bash
   chmod +x load_test_start.sh load_test.sh load_test_stop.sh get_conf_param.sh get_vm_values.sh
   ```
3. Создайте служебную базу данных `expecto_db` и пользователя `expecto_user`:
   ```sql
   CREATE DATABASE expecto_db;
   CREATE USER expecto_user WITH PASSWORD '...';
   GRANT ALL PRIVILEGES ON DATABASE expecto_db TO expecto_user;
   ```
4. Создайте необходимые таблицы и функции в `expecto_db` (см. раздел [Структура expecto_db](#структура-expecto_db)).
5. Настройте `param.conf` (см. ниже).

---

## ⚙️ Конфигурация (`param.conf`)

| Параметр | Описание | Пример |
|----------|----------|--------|
| `start_load` | Начальное количество клиентов pgbench | `5` |
| `finish_load` | Максимальное количество клиентов | `20` |
| `testdb` | Тестовая БД: `default` (создаётся `pgbench_db`) или имя существующей БД | `default` |
| `load_mode` | Тип сценариев: `oltp` или `olap` | `olap` |
| `scenario1`, `scenario2`, `scenario3` | Веса сценариев (доли от общего числа клиентов) | `0.7`, `0.2`, `0.1` |
| `init_test_db` | При `on` – БД будет создана и наполнена заново, иначе – используется существующая | `on` |
| `scale` | Масштаб pgbench (количество строк в таблицах) | `685` (≈10 ГБ) |

> **Примечание:** если `testdb` не равен `default`, то предполагается, что БД уже существует, и `init_test_db` игнорируется. В этом случае владельцем БД должен быть пользователь, указанный в `load_test_start.sh` (переменная `testdb_owner`).

## 🚀 Запуск и остановка теста

### 1. Инициализация теста
```bash
./load_test_start.sh
```
Скрипт:
- Создаёт новую запись в `expecto_db`.
- При необходимости инициализирует тестовую БД (`pgbench -i`).
- Применяет `before_start.sql`.
- Создаёт функции сценариев (из файлов `.oltp.sql` или `.olap.sql`).
- Генерирует `do_scenario*.sql`.
- Сохраняет текущие параметры ядра в `expecto_db`.
- Создаёт файл-флаг `LOAD_TEST_STARTED`.

### 2. Запуск итераций
Настройте cron для регулярного вызова (например, раз в минуту):
```cron
* * * * * /postgres/pg_expecto/load_test/load_test.sh
```
`load_test.sh`:
- Проверяет наличие флага `LOAD_TEST_STARTED`.
- Если итерация не выполняется – запускает новую:
  - Определяет текущий проход (нагрузку) и число клиентов.
  - Запускает `pgbench` для каждого сценария с соответствующим числом клиентов.
  - Собирает метрики (вызовы функций `load_test_start_collect_data` / `load_test_stop_collect_data`).
- По достижении максимальной нагрузки (`finish_load`) автоматически завершает тест.

### 3. Остановка теста
```bash
./load_test_stop.sh
```
Скрипт:
- Удаляет все флаги.
- Останавливает сбор метрик.
- Завершает все процессы `pgbench` и выполняющиеся сценарии.

## 📊 Сбор данных

Система предполагает наличие в `expecto_db` следующих функций и таблиц (не входят в состав поставки). Примерный список:

| Функция | Назначение |
|---------|------------|
| `load_test_new_test()` | Начать новый тест, вернуть `test_id`. |
| `load_test_set_start_load(int)` | Задать начальную нагрузку. |
| `load_test_set_max_load(int)` | Задать максимальную нагрузку. |
| `load_test_current_pass()` | Вернуть номер текущего прохода (нагрузки). |
| `load_test_set_load()` | Установить следующую нагрузку, вернуть число клиентов. |
| `load_test_get_load()` | Получить число клиентов для текущего прохода. |
| `load_test_set_weight_for_scenario(int, float)` | Установить вес сценария. |
| `load_test_get_load_by_scenario(int)` | Получить число клиентов для конкретного сценария (с учётом веса). |
| `load_test_start_collect_data()` | Начать сбор метрик. |
| `load_test_stop_collect_data()` | Остановить сбор метрик. |
| `load_test_set_scenario_queryid()` | Сохранить планы запросов (Query ID). |
| `load_test_is_test_could_be_finished()` | Проверить, достигнута ли максимальная нагрузка. |
| `load_test_get_current_test_id()` | Получить текущий `test_id`. |
| `save_dirty_background_ratio(int)` и аналогичные | Сохранить параметры ядра. |

## 📝 Примечания

- При использовании режима `default` БД `pgbench_db` создаётся и наполняется данными `pgbench` с указанным масштабом.
- Скрипт `before_start.sql` добавляет в таблицу `pgbench_history` столбец `random_fill` и индекс на него – это необходимо для корректной работы некоторых сценариев (например, `scenario2.oltp.sql` использует этот столбец).
- Для кастомной тестовой БД (`testdb != default`) предполагается, что функции сценариев уже существуют или их нужно создать из файлов `scenarioN.sql` (без суффикса `.oltp`/`.olap`). В текущей версии скрипт использует `scenarioN.sql` (общий) – это можно адаптировать при необходимости.
- Флаги `LOAD_TEST_STARTED`, `LOAD_TEST_IN_PROGRESS`, `PGBENCH_WORKING` создаются в каталоге скриптов и используются для синхронизации и предотвращения одновременных запусков.
- Все логи пишутся в файлы `load_test.log` и `load_test.err` в том же каталоге.

## 🐛 Устранение неполадок

- **Тест не запускается** – проверьте наличие флага `LOAD_TEST_STARTED`, права на создание/запись в каталог.
- **Ошибка `psql: FATAL: database "expecto_db" does not exist`** – создайте БД и пользователя.
- **Нет функций в expecto_db** – создайте их вручную или обратитесь к разработчику.
- **pgbench не запускается** – проверьте, что переменные окружения PostgreSQL настроены и `pgbench` доступен из PATH.
- **Сценарии не создаются** – убедитесь, что в каталоге есть файлы `scenarioN.oltp.sql` или `scenarioN.olap.sql` в зависимости от `load_mode`.
