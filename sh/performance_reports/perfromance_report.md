# Генерация отчёта по производительности СУБД и ОС с использованием DeepSeek

Данный документ содержит пошаговую инструкцию по подготовке аналитического отчёта на основе результатов нагрузочного тестирования СУБД PostgreSQL. Отчёт формируется с привлечением методологии философского анализа и языковой модели DeepSeek.

## Важные условия

- Скрипт `performance_report.sh` установлен по фиксированному абсолютному пути:
  ```
  /postgres/pg_expecto/sh/performance_reports/performance_report.sh
  ```
  Изменение пути при инсталляции не предусмотрено.

- Архив `performance_4deepseek.zip` формируется на первом шаге и **не загружается** в репозиторий.

- Взаимодействие с DeepSeek производится **вручную** через веб-интерфейс [chat.deepseek.com](https://chat.deepseek.com). Автоматизация не предусмотрена.

## Порядок действий

### 1. Сформировать первичный отчёт по производительности СУБД и ОС

Выполните команду:

Входные параметры: начальное и конечное время отрезка сбора данных производительности СУБД и ОС

```bash
/postgres/pg_expecto/sh/performance_reports/performance_report.sh '2026-04-13 09:47' '2026-04-13 12:13'
```

В результате работы скрипта будет создан архив `performance_4deepseek.zip`, содержащий исходные файлы для дальнейшего анализа.

### 2. Подготовить целевую папку и распаковать архив

Создайте отдельную папку (далее — целевая) и распакуйте в неё полученный архив:

```bash
mkdir -p ./deepseek_workspace
unzip performance_4deepseek.zip -d ./deepseek_workspace
cd ./deepseek_workspace
```

### 3. Визуально проверить состав файлов

Убедитесь, что в целевой папке присутствуют все пять файлов:

- `_1.settings.txt` — конфигурация СУБД и ОС
- `_2.postgresql_vmstat_iostat.txt` — статистические данные производительности СУБД и ОС
- `pg_expecto_instruction.txt` — доменная и философская инструкция инструкция нейросети по подготовке отчёта
- `prompt_source.txt` — промпт для подготовки сводного отчета отчёта по нагрузочному тестированию
- `prompt_result.txt` — промпт для подготовки аналитического отчёта по нагрузочному тестированию

Для проверки можно использовать команду `ls -1` или открыть папку в файловом менеджере.

### 4. Подготовить сводный  отчёт `source.txt`

1. Откройте новый чат в [DeepSeek](https://chat.deepseek.com).
2. Включите **«Глубокое мышление»** и **«Умный поиск»**.
3. Загрузите файл `_pg_expecto_instruction.txt`.
4. Введите сообщение:
   ```
   Строго следуй инструкции, ожидай входные данные.
   ```
5. Загрузите файлы `_1.settings.txt` и `_2.postgresql_vmstat_iostat.txt`.
6. Введите в качестве следующего сообщения полное содержимое файла `prompt_source.txt`.
7. Запустите генерацию ответа.
8. Полученный результат сохраните в текстовый файл `source.txt` в целевой папке.

### 5. Подготовить финальный аналитический отчёт `result.txt`

1. Начните **новый чат** в DeepSeek.
2. Включите **«Глубокое мышление»** и **«Умный поиск»**.
3. Загрузите файл `pg_expecto_instruction.txt`.
4. Введите сообщение:
   ```
   Строго следуй инструкции, ожидай входные данные.
   ```
5. Загрузите файл `source.txt`.
6. Введите в качестве сообщения полное содержимое файла `prompt_result.txt`.
7. Запустите генерацию ответа.
8. Сохраните итоговый результат как финальный аналитический отчёт (например, `result.md` или `result.txt`).

## Результат

В целевой папке будут находиться:

- `source.txt` — сводный  отчёт по нагрузочному тестированию.
- `result.txt` - Аналитический отчёт по нагрузочному тестированию.

## Примечания

- Все действия, связанные с DeepSeek, выполняются вручную. Скрипты автоматизации не предусмотрены.
- Архив `perfromace_4deepseek.zip` не должен добавляться в систему контроля версий.
- Путь к скрипту `performance_report.sh` жёстко задан и не требует изменения.
