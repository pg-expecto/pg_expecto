## Назначение

Скрипт собирает несколько SQL-файлов в один результирующий файл `pg_expecto.sql`, удаляет из них строки с версионными комментариями (содержащие слово `version` в однострочном SQL-комментарии), добавляет в начало лицензию из отдельного файла и копирует итоговый файл в `/tmp` с правами доступа `777`.

Используется в процессе подготовки пакета `pg_expecto` для распространения единого SQL-скрипта.

## Требования

- ОС: Linux / macOS / WSL (или любая среда с `bash`)
- Утилиты: `bash`, `sed`, `cat`, `cp`, `chmod`, `mktemp`
- Права: запись в текущую директорию и в `/tmp`

## Подготовка

Перед запуском убедитесь, что в **текущей рабочей директории** присутствуют следующие файлы:

| Файл               | Описание                                                                 |
|--------------------|--------------------------------------------------------------------------|
| `sql_source.txt`   | Список SQL-файлов для объединения (по одному относительному пути на строку). Пустые строки и строки, начинающиеся с `#`, игнорируются. |
| `license.txt`      | Текст лицензии, который будет добавлен в начало итогового файла.         |
| Исходные `.sql`    | Файлы, перечисленные в `sql_source.txt` (должны существовать и быть доступны для чтения). |

Пример содержимого `sql_source.txt`:
```text
# Основные компоненты
core/types.sql
core/functions.sql
# Опционально:
extras/helpers.sql
```

## Использование

```bash
# Сделать скрипт исполняемым (однократно)
chmod +x prepare_pg_expecto_sql.sh

# Запустить
./prepare_pg_expecto_sql.sh

## Что делает скрипт (пошагово)

1. Проверяет наличие файлов `sql_source.txt` и `license.txt`. Если хотя бы одного нет – завершается с ошибкой.
2. Читает `sql_source.txt`, игнорируя пустые строки и комментарии.
3. Для каждого указанного SQL-файла:
   - Проверяет существование и доступность для чтения.
   - Добавляет содержимое файла во временный сборщик, вставляя после каждого файла пустую строку (чтобы избежать склейки кода).
4. Удаляет из собранного текста **целые строки**, которые содержат:
   - однострочный SQL-комментарий (`--`) **и** слово `version` (без учёта регистра).
   - Пример удаляемой строки: `-- version 1.2.3` или `-- this is version info`.
5. Добавляет содержимое `license.txt` в самое начало результирующего файла.
6. Сохраняет результат как `pg_expecto.sql` в текущей директории.
7. Копирует `pg_expecto.sql` в `/tmp/` с именем `pg_expecto.sql`.
8. Устанавливает права `777` на файл в `/tmp/` (чтение/запись/исполнение для всех пользователей).

## Результат

- `pg_expecto.sql` – собранный и очищенный SQL-файл в текущей папке.
- `/tmp/pg_expecto.sql` – копия с правами `777`.

## Пример вывода в терминале

```text
1. Чтение списка файлов из 'sql_source.txt'...
   Добавление: core/types.sql
   Добавление: core/functions.sql
2. Сформирован временный файл со всем содержимым.
3. Удаление строк комментариев с 'version'...
4. Добавление содержимого 'license.txt' в начало...
5. Копирование 'pg_expecto.sql' в /tmp/...
6. Установка прав 777 на /tmp/pg_expecto.sql...
Готово. Результирующий файл: /tmp/pg_expecto.sql
```

## Примечания

- Многострочные комментарии (`/* ... */`) не обрабатываются – удаление `version` работает только для строк, начинающихся с `--`.
- Если какой-либо файл из списка отсутствует или недоступен, скрипт выдаст предупреждение и продолжит работу с остальными.
- Скрипт использует `set -euo pipefail`, поэтому любая ошибка (например, отсутствие `sed`) приведёт к немедленной остановке.
- Временные файлы автоматически удаляются при завершении скрипта (даже при ошибке).

## Возможные расширения (не реализованы в текущей версии)

- Удаление только слова `version` внутри комментария, а не всей строки.
- Поддержка многострочных комментариев.
- Переопределение имени выходного файла через аргумент командной строки.
